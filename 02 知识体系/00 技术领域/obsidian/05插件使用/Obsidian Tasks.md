Obsidian Tasks 插件中，任务文本后的 emoji 图标是「元数据标记」，用于定义任务属性、驱动查询与视图渲染，以下是核心图标（含默认符号）的作用、用法与示例，覆盖优先级、日期、重复等高频场景。

---

### 二、详细作用与操作指南
#### 1. 优先级图标（🔺/🔼/🔽）
- 作用：定义任务重要程度，影响任务查询的排序规则与视觉权重。
- 操作：
  1. 任务文本后直接输入 emoji（🔺/🔼/🔽）。
  2. 命令面板执行「Tasks: Edit Task」，在弹窗中选择优先级。
- 关键：优先级无强制约束，可通过 CSS 自定义颜色/样式，适配个人管理习惯。

#### 2. 日期类图标（📅/⏳/🛫/✅）
- 📅 截止日期（due）：
  - 作用：设定任务最后完成期限，过期任务会在视图中高亮提醒。
  - 语法：📅 后接 ISO 日期（YYYY-MM-DD），支持自然语言解析（如 📅 tomorrow）。
- ⏳/🛫 计划开始（scheduled/start）：
  - 作用：设定任务开始处理时间，未到时间的任务默认不显示在“今日任务”视图。
  - 语法：⏳ 2026-01-30 或 🛫 2026-01-30，两者功能一致。
- ✅ 完成日期（done）：
  - 作用：自动记录任务完成时间，用于复盘统计（如“本周完成任务”查询）。
  - 操作：勾选任务完成后，插件自动添加 ✅ 及当前日期，无需手动输入。

#### 3. 重复规则图标（🔁）
- 作用：设定任务重复周期，完成后自动生成下一次任务，适配周期性工作（如日报、周会）。
- 语法：🔁 后接重复规则，支持自然语言与 CRON 表达式。
  - 示例 1（自然语言）：- [ ] 每日打卡 🔁 every day
  - 示例 2（CRON）：- [ ] 每周备份 🔁 0 0 * * 6（每周六执行）
- 关键：重复任务完成后，插件会在原始笔记中更新下次任务的日期，保持视图动态同步。

#### 4. 其他常用图标（自定义扩展）
- 🔻 无优先级：用于标记非核心任务，在查询中可通过「priority is low」过滤。
- 🔴 阻塞（blocked）：自定义状态图标，标记依赖未完成的任务，语法示例：- [ ] 开发功能 🔴 blocked（需在插件设置中配置自定义状态）。

### 三、使用技巧与注意事项
1. **快速添加图标**：
   - 命令面板执行「Tasks: Create or Edit Task」，在弹窗中一键设置优先级、日期、重复规则，自动生成对应图标。
   - 绑定快捷键（如 Ctrl+T），快速调出任务编辑弹窗，提升操作效率。
2. **查询与视图联动**：
   - 任务查询代码块中，通过「priority is high」「due before tomorrow」等条件筛选图标标记的任务。
   - 示例（查询今日截止的高优先级任务）：
     ```tasks
     not done
     priority is high
     due today
     ```
3. **自定义扩展**：
   - 插件设置中可修改优先级图标、日期格式、重复规则默认值。
   - 通过 CSS 片段自定义图标颜色与大小，如：
     ```css
     .task-priority-high { color: #ff5252; } /* 🔺 高优先级红色 */
     .task-due-soon { color: #ff9100; } /* 即将过期任务橙色 */
     ```

### 四、常见问题排查
1. 图标不生效：检查是否启用 Tasks 插件，确保语法格式正确（图标与任务文本间无多余空格）。
2. 重复任务不生成：确认重复规则语法正确，完成任务时勾选「Mark as done and create next occurrence」。
3. 日期解析失败：优先使用 ISO 格式（YYYY-MM-DD），避免自然语言歧义（如“next week”需明确起始时间）。
