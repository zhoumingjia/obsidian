# 6 款 Obsidian 插件快速配置清单
（适配 Java 后端开发 / 技术文档 / 项目管理工作流）

## 一、QuickAdd（自动化流程）

### 核心设置

1. 打开插件设置 → **Add Choice**，命名`新建接口文档`
2. 选择类型`Template`，关联 Templater 模板文件（路径示例：`Templates/接口文档模板.md`）
3. 勾选`Open after creation`，设置快捷键`Ctrl+Shift+I`

### 常用脚本（一键生成接口参数）

```javascript
// 自动填充Java接口基础结构
const content = `---
title: 接口-${tp.date.now("YYYYMMDDHHmmss")}
模块: 双录系统/音视频集群
作者: 
创建时间: ${tp.date.now("YYYY-MM-DD HH:mm")}
---
## 接口信息
- 接口路径: /api/
- 请求方式: POST/GET
- 入参类型: application/json
- 出参类型: application/json

## 入参示例
\`\`\`java
// 示例POJO类
public class RequestDTO {
    private String id; // 唯一标识
}
\`\`\`

## 出参示例
\`\`\`java
public class ResponseDTO {
    private int code; // 响应码
    private String msg; // 响应信息
}
\`\`\`
`;
await app.vault.create(`接口文档/接口-${tp.date.now("YYYYMMDDHHmmss")}.md`, content);
```

### 避坑要点

- 脚本中路径需用`/`分隔，避免中文路径
- 联动 Templater 时，需确保 Templater 插件已启用

## 二、Obsidian Git（版本管理）

### 核心设置

1. 初始化仓库：点击插件图标 → **Initialize Repository**
2. 自动提交配置：
    
    - 勾选`Enable auto commit`，间隔设为`30分钟`
    - 勾选`Enable auto push`，勾选`Pull before push`
    
3. 忽略文件配置：新建`.gitignore`文件，添加内容
    
    plaintext
    
    ```
    .obsidian/workspace.json
    .obsidian/plugins/
    *.log
    ```

### 常用快捷键

- `Ctrl+P` → 输入`Git: Commit`：手动提交
- `Ctrl+Shift+P` → 输入`Git: Show History`：查看版本历史

### 避坑要点

- 提交前确保 Vault 路径无空格 / 特殊字符
- 多人协作时，先`Pull`再提交，避免冲突

## 三、Codeblock Customizer（代码块美化）

### 核心设置

1. 启用插件 → 进入设置，开启以下选项：
    
    - `Show line number`：显示行号
    - `Show language label`：显示代码语言标识
    - `Enable copy button`：添加复制按钮
    - `Enable code folding`：支持代码块折叠
    
2. 自定义 Java 代码块样式：
    
    - 背景色：`#f5f5f5`
    - 行号颜色：`#999999`
    - 高亮行颜色：`#e6f7ff`
    

### 避坑要点

- 样式修改后，需重启 Obsidian 生效
- 与主题冲突时，关闭主题自带的代码块样式

## 四、Excalibrain（知识图谱可视化）

### 核心设置

1. 启用插件 → 快捷键`Ctrl+Shift+E`打开图谱
2. 节点筛选：
    
    - 勾选`Only show nodes with tag`，输入`#系统架构/#接口设计`
    - 节点大小：按`文件链接数`排序
    
3. 样式配置：
    
    - 线条颜色：`#4299e1`
    - 节点颜色：`#38b2ac`（架构类笔记）、`#9f7aea`（接口类笔记）
    

### 适配场景

- 生成双录系统架构图谱：标签筛选`#双录系统`，自动关联子模块笔记

### 避坑要点

- 笔记数量过多时，关闭`Show all connections`，避免卡顿
- 依赖笔记间的双向链接，需确保文档链接规范

## 五、Linter（文档规范化）

### 核心规则配置

1. 启用插件 → 进入设置，开启以下规则：
    
    - `Headers: Consistent header hierarchy`：统一标题层级（# 一级，## 二级）
    - `Lists: Proper list indentation`：列表自动缩进
    - `YAML: Ensure YAML front matter exists`：强制添加 YAML 元数据
    - `Whitespace: Remove trailing whitespace`：删除行尾空格
    
2. 快捷键：`Ctrl+Alt+L` 格式化当前笔记

### 避坑要点

- 自定义规则时，排除`代码块`与`表格`，避免格式错乱
- 团队协作时，导出规则文件共享，确保格式统一

## 六、Kanban（看板任务管理）

### 核心设置

1. 新建看板：右键 → `New Kanban Board`，命名`双录系统迭代看板`
2. 配置列：`待开发 → 开发中 → 测试 → 上线 → 归档`
3. 任务卡片格式：
    
    markdown
    
    ```
    - [ ] 【接口开发】新增音视频上传接口 #后端 #高优
      - 截止日期：2026-XX-XX
      - 关联文档：[[接口-20260116153000]]
    ```
    
4. 联动 Tasks 插件：看板任务自动同步到 Tasks 查询列表

### 避坑要点

- 看板文件后缀为`.kanban`，不要修改为`.md`
- 任务标签与 Dataview 联动，可实现跨看板筛选
